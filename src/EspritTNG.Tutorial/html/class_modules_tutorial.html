Introduction to Class Modules

This tutorial introduces the user-defined classes of objects.

The class defines the things that those types of objects have in common. The libraries that form the ESPRIT API provide many predefined classes, but you are not limited to create the new ones.

Like other objects, the ones you define with class modules can have Properties, Methods, and Events.

As well as form modules, class modules respond to events. These events are triggered by certain conditions or user interaction (such as the clicking of a command button) and they could be executed in any order, any number of times (or possibly not at all). Classes and forms must be initialized, and they can be terminated. The way that a form is initialized and terminated is with the Load and Unload commands, as is shown in the various tutorials using forms, like Tutorial: Introduction to Using UserForms. The way that a class module is initialized is the same as other objects. The way that that a class module is terminated is also the same as other objects, it can be set to null, or it can simply go out of scope.

Define a class with initialize and terminate methods:

C#
class ClassModulesHandler : IDisposable
{
public ClassModulesHandler()
{
MessageBox.Show("This is the Constructor procedure.", "ClassModulesHandler", MessageBoxButton.OK);
}
public void Dispose()
{
MessageBox.Show("This is the Terminate procedure.", "ClassModulesHandler", MessageBoxButton.OK);
}
}

Now we can create a class object and get immediately the initialize message and a terminate messages right after. Class object goes out of scope just after the execution procedure ends.

C#
public void ClassExample1()
{
ClassModulesHandler testObject = new ClassModulesHandler();
testObject.Dispose();
}
